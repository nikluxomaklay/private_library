## User Scenarios & Testing

### User Story 1 - Просмотр детальной информации о Reading log (Priority: P1)

**Как пользователь системы**, я хочу иметь возможность просматривать детальную информацию о Reading log на отдельной странице.

**Почему этот приоритет**: Это базовая функция для навигации по системе. Пользователь должен иметь возможность быстро перейти к подробной информации о любом Reading log из различных мест интерфейса. Без этой функции редактирование невозможно.

**Независимый тест**: Пользователь может перейти на детальную страницу Reading log из Home, списка Reading log или страницы Book edition и увидеть всю информацию в режиме только для чтения.

**Сценарии принятия**:

1. **Given** пользователь находится на странице Home, **When** кликает на период Reading log, **Then** открывается детальная страница Reading log в режиме read-only
2. **Given** пользователь находится в списке Reading log, **When** кликает на период Reading log, **Then** открывается детальная страница Reading log в режиме read-only
3. **Given** пользователь находится на странице Book edition, **When** кликает на период Reading log, **Then** открывается детальная страница Reading log в режиме read-only
4. **Given** пользователь на детальной странице Reading log, **When** просматривает поля, **Then** все поля отображаются в режиме read-only (аналогично странице создания)

---

### User Story 2 - Редактирование Reading log (Priority: P2)

**Как пользователь системы**, я хочу иметь возможность редактировать Reading log для исправления или обновления информации о периоде чтения.

**Почему этот приоритет**: Это основная ценность функции. После просмотра деталей пользователь может изменить период чтения. Функция зависит от User Story 1, так как переход к редактированию осуществляется с детальной страницы.

**Независимый тест**: Пользователь может перейти с детальной страницы на страницу редактирования, изменить поля Year start, Month start, Year finish, Month finish и сохранить изменения.

**Сценарии принятия**:

1. **Given** пользователь на детальной странице Reading log, **When** кликает кнопку "Update" в блоке действий, **Then** открывается страница редактирования с формой, аналогичной странице создания
2. **Given** пользователь на странице редактирования, **When** изменяет поля Year start, Month start, Year finish, Month finish, **Then** изменения сохраняются при подтверждении
3. **Given** пользователь на странице редактирования, **When** выбирает пустое значение для любого из полей, **Then** поле успешно очищается и сохраняется
4. **Given** пользователь внёс изменения, **When** подтверждает сохранение, **Then** перенаправляется на детальную страницу с обновлёнными данными
5. **Given** пользователь на странице редактирования, **When** нажимает кнопку отмены, **Then** перенаправляется на детальную страницу Reading log без сохранения изменений

---

### Edge Cases

- Reading log МОЖЕТ существовать без указанного периода (все поля пустые)
- Система ДОЛЖНА блокировать сохранение при попытке установить дату окончания раньше даты начала с отображением сообщения об ошибке
- При редактировании Reading log, связанного с книгой с другими записями чтения, изменения применяются только к текущей записи

## Requirements

### Functional Requirements

- **FR-001**: Система ДОЛЖНА предоставлять детальную страницу просмотра для каждого Reading log
- **FR-002**: Система ДОЛЖНА отображать период Reading log как ссылку на детальную страницу на странице Home
- **FR-003**: Система ДОЛЖНА отображать период Reading log как ссылку на детальную страницу в списке Reading log
- **FR-004**: Система ДОЛЖНА отображать период Reading log как ссылку на детальную страницу на странице Book edition (вместо текущей ссылки на год чтения)
- **FR-005**: Детальная страница Reading log ДОЛЖНА отображать все поля в режиме read-only (аналогично странице создания)
- **FR-006**: Система ДОЛЖНА предоставлять кнопку "Update" в блоке действий на детальной странице Reading log
- **FR-007**: Страница редактирования Reading log ДОЛЖНА иметь форму, аналогичную странице создания
- **FR-008**: Система ДОЛЖНА позволять редактировать поля: Year start, Month start, Year finish, Month finish
- **FR-009**: Система ДОЛЖНА позволять устанавливать пустое значение для всех редактируемых полей
- **FR-010**: Система ДОЛЖНА сохранять изменения после подтверждения редактирования
- **FR-011**: Система ДОЛЖНА блокировать сохранение при попытке установить дату окончания раньше даты начала с отображением сообщения об ошибке
- **FR-012**: Reading log МОЖЕТ существовать без указанного периода (все поля Year start, Month start, Year finish, Month finish могут быть пустыми)
- **FR-013**: Система ДОЛЖНА предоставлять кнопку отмены на странице редактирования для возврата на детальную страницу без сохранения

### Key Entities

- **Reading log**: Запись, представляющая период чтения книги. Содержит поля: Year start, Month start, Year finish, Month finish, и связана с Book edition.

## Success Criteria

### Measurable Outcomes

- **SC-001**: Пользователь может перейти на детальную страницу Reading log из любого места (Home, список Reading log, Book edition) за 1 клик
- **SC-002**: Пользователь может отредактировать все 4 поля (Year start, Month start, Year finish, Month finish) и сохранить изменения в течение 1 минуты
- **SC-003**: 100% полей, доступных для редактирования, поддерживают установку пустого значения
- **SC-004**: После редактирования пользователь автоматически перенаправляется на обновлённую детальную страницу
